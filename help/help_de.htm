Audio_XH - ein CMSimple_XH-Plugin

Audio_XH ermöglicht die Darstellung von Audio-Playern auf einer CMSimple_XH Website. Sie müssen eine MP3 und eine Ogg Vorbis Datei für jeden Player bereit stellen. Wenn der Browser des Besuchers das AUDIO element unterstützt, nutzt das Plugin dieses Feature, andernfalls fällt es auf einen einfachen Flash-Player zurück. Wenn nicht einmal Flash verfügbar ist, dann wird ein Link zur MP3 Datei angezeigt. JavaScript wird nicht benötigt.

Requirements

Audio_XH ist ein Plugin für CMSimple_XH. Es benötigt eine UTF-8 kodierte Version.

Installation

Die Installation erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch. Im CMSimple_XH-Wiki finden Sie weitere Details.

  1. Sichern Sie die Daten auf Ihrem Server.
  2. Entpacken Sie die ZIP-Datei auf Ihrem Rechner.
  3. Laden Sie das ganze Verzeichnis audio/ auf Ihren Server in CMSimple_XHs Plugin-Verzeichnis hoch.

Verwendung

Da Audio_XH keine solche Möglichkeit anbietet, müssen Sie Ihre Audio-Dateien in den konfigurierten Media-Ordner per FTP, dem Filebrowser oder Uploader_XH hoch laden. Jede Audiodatei muss als MP3 (.mp3) und als Ogg Vorbis (.oga) Version hoch geladen werden; beide Dateien müssen im selben Ordner sein und den selben Namen haben. Um eine Audiodatei in die benötigten Formate zu konvertieren, können Sie encode4web verwenden.

Um einen Audio-Player auf einer Seite darzustellen, fügen Sie einfach den folgenden Pluginaufruf ein:

{{{PLUGIN:audio('DATEINAME');}}}

DATEINAME muss ein Dateipfad (ohne Dateieerweiterung) relativ zum konfigurierten Media Ordner sein.

Wenn Sie also zum Beispiel musik.mp3 und musik.oga direkt im Media-Ordner abgelegt haben, verwenden Sie:

{{{PLUGIN:audio('musik');}}}

Um einen Audio-Player auf allen Seiten darzustellen, fügen Sie einfach folgendes in Ihr Template ein:

<?php echo audio('DATEINAME');?>
